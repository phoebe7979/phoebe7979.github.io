THE CHALLENGE

How might we help code enforcement officers leverage Dynamo Metrics' data power for their daily inspection work?


As a data-visualization and consulting company, Dynamo Metrics' has been in the making of a tailor-made data visualization application for the code enforcement team at the City of Jackson. With little UX resources, the company was in need of professioal insights to deliver an optimal solution. Our team was tasked with understanding user needs and translating them into design solutions for the application.

Our team was tasked to help our client design a platform for the City of Jackson.

Insights generated from multiple research methods

Over the course of heuristic evaluation, field research, observation and interviews, our findings can be categorized into four key areas.

THE SOLUTION: NEIGHBORHOOD DATA

Streamlined experinece viewing urban data on a single web application

I was solely responsible for redesigning the neighborhood data feature of the application. By keeping only the most important elements while adding necessary functionality based on the results of research, I created a series of mockups and interactions for the redesigned UI.



Neighborhood Summary

Pain Point
Officers switched between different pages on their previous platform, and rely on other department's analytics to aggregate neightborhood data. They could not quickly prioritize patroling resource.
Solution
Officers can check instant summary of an area on the map. A newly added 'changeability index' notifies officers of how easy an area's code violations can be effectively reduced. This can help the code enforcement team allocate thei resources efficiently.



A neeighborhood summary panel shows up when the user mouses over or taps on an area on the map.


New Layer Adding Process

Pain Point
The old UI from our client requires the user to add data layer by selecting from a long list of checkboxes. Users have to manually uncheck and recheck an item in order to move a layer to the top.
Solution
Data types are added with a separated layer building process. Data layers are now draggable for reorder, with feature to toggle visibility for quick data interpretation.



The old layer adding menu includes a long scrolling checklist, without a clear display of layer adding status.

The redesigned payer adding interaction. Users can now clearly see how many layers have been applied, and can also manipulate with different layers when interpreting data.
