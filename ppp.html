THE PROBLEM

How can the client's existing technology meet code enforcement officers' needs?


For a semester-based extracurricular UX consulting initiative, I worked in a team of two data scientists and three UX designers to enhance the Dynamo Metrics, a data visualization company.

The company's prospecitve project is a tailor-made data visualization application for the code enforcement team at the City of Jackson. With little UX background and resource, the company is in need of professioal insights to deliver optimal solution. Our deliverables were to conduct user research and provide a mid-fi mockup




THE MOCKUP - MY DESIGN

Keeping only 1/10 of visual elements on the 'neighborhood data' interface


After a series of user research, I am solely responsible for redesigning the neighborhood data feature of the application. By keeping only the most important elements, I created several mockup pages that illustrate how the new design directs users to the main focus -- the data. Below are the 5 key revisions of the application that I contributed:


Neighborhood Data On Hover

Check out instant summary of neighborhood data when hovering over corresponding area on map. An addition of changeability index in the statistics allows for instant target identification.

New Layer Adding Process

Reduced visual distraction on UI, presenting only necessary information.

Draggable Layer Design

Added drag-and-drop features for each layers of data on the map for intuitive navigation and data interpretation.

Legend Panel on Map

Added legend panel with collapsible details on the map for more efficient interpretation.

Redesigned Historic Trend Playback

Rearranged configurations for more streamlined experience.

THE PROCESS

From zero to final pitch



The team has communicated effectively with client upfront to learn their expectations. UX RESEARCH

What exactly is code enforcement? Understanding users from a completely unfamiliar field.


The end users for this projects are very simple -- the Code Enforcement Department of the City of Jackson Government. My teammates and I took several steps to learn about who the users really are, as well as their needs.



Background Reserach


Without much prior knowledge of both our clients and the users, the whole team conducted a throrough background research by researching the client's past projects, inspecting current software interface, while reading governmental webpages and documents to understand the Code Enforcement Department's various code ordinances they are responsible for.


"A Code Enforcement Officer is a ... inspector, ... ,employed by a city,..., whose primary duties are the prevention, detection, investigation, and enforcement of violations of laws regulating public nuisance, public health, safety, and welfare, public works, business activities and consumer protection, building standards, land-use, or municipal affairs."1

Aside from understanding related ordinnces, we especially paid attention to the common complaints and violations that code enforcement officers often have to deal with.



Competitive Analysis

We then proceeded with competitive analysis with five code enforcement related solutions and one other data visuazliation application, for which we examined the following based on public information on their websites:


  • Platform & supported devices
  • Data types
  • Key workflow
  • UI elements

The results showed that most applications are browser-friendly, but whether the content is mobile friendly is unknown (available information all showed desktop view). Considering officers may require access to certain data during their missions on-site, having a mobilt-friendly solution may be useful. In addition, some competitors provide summative data or index. Our client's current application would improve by adding predictive and summative data for users to quickly learn the status of an area.

As for key workflow, most competitors offer productivity and administrative features aside from data visualization tasks so the users can complete most tasks including issuing permits, file complaints, reporting blights, and update property status; on the other hand, our client's solution specifically focuses on the presentation of city data and only includes functionality to investigate different types of data.

As for visual presentation and interaction experience, our clients' curernt design incorporates multiple accordions and checkboxes, which result in much visual burden and interaction cost, while wasting quite some space. On the other hand, two competitors utilized floating blocks or hidden features to solve the space issue, providing more real estate for the focus of the application -- the visual presentation of the data.



User Interviews

Four interviews were conducted at the City of Jackson with potential users in the government. Prior to the interview, the whole team worked together on a semi-structured interview protocol with an aim to elicit specific details regarding the interviewee's background, job responsibilities, daily workflow, challenges, and expectations.



After the interview, the UX team interpreted the data and organized the notes into a structured affinity wall, including:


  • Daily tasks
  • Duties
  • Issues regarding workflow
  • Issues regarding software tools
  • Data types in need


Personas

After the interviews, each UX designer developed a persona, each representing a type of the end users: code enforcement director, code enforcement officer, and administrator.



Workflow

Prior to the design phase, the UX team developed a flowchart that details the daily routine of the end users. With the help of the flow chart, we were able to efficiently design an interface collaborately.



THE RESULTS

The presentation day and an opportunity


At the end of the semester project, we presented the results in front of our clients at an exposition. The client was very happy about our redesign and provided some ideas on the prototype for future revisions.



I was also offered a semester-long internship opportunity to continue the user testing and revisions with another UX designer in the team.
